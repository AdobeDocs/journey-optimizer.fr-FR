# Interface utilisateur {#cjm-user-interface}

![](assets/do-not-localize/badge.png)

Cette documentation est fréquemment mise à jour pour refléter les modifications récentes du produit. Cependant, certaines captures d’écran peuvent différer légèrement de votre interface utilisateur.

## Espace de travail {#cjm-workspace}

Une fois connecté à [Adobe Experience Cloud](http://experience.adobe.com), accédez à [!DNL Journey Optimizer].

>[!NOTE]
>
>Les concepts clés lors de la navigation dans l’interface utilisateur sont détaillés dans [la documentation de Adobe Experience Platform](https://experienceleague.adobe.com/docs/experience-platform/landing/platform-ui/ui-guide.html?lang=en#adobe-experience-platform-ui-guide).

Utilisez les liens sur la gauche pour parcourir les fonctionnalités.

![](assets/ajo-home.png)

>[!NOTE]
>
>Les fonctionnalités disponibles peuvent varier en fonction de vos autorisations et de votre contrat de licence.

Vous trouverez ci-dessous la liste complète des entrées dans le rail de gauche et des liens vers la documentation associée.

* **[!UICONTROL Surveillance]**  : utilisez ce menu pour surveiller l&#39;assimilation de vos données dans l&#39;interface utilisateur de Adobe Experience Platform. ## Prise en charge des navigateurs et des langues

L&#39;interface d&#39;Adobe [!DNL Journey Optimizer] est conçue pour fonctionner de manière optimale dans la dernière version de Google Chrome. Vous pouvez rencontrer des problèmes lors de l’utilisation de certaines fonctions sur des versions plus anciennes ou d’autres navigateurs.

L’interface utilisateur est actuellement disponible dans les langues suivantes :

* Anglais
* Français
* Allemand

Votre langue d’interface par défaut est déterminée par la langue préférée spécifiée dans votre profil d’utilisateurs.

Pour modifier votre langue :

* Cliquez sur **Modifier les préférences** dans votre avatar, en haut à droite.
* Cliquez ensuite sur **Visiter le compte d’Adobe** pour accéder à votre profil d’Adobe.
* Sélectionnez l&#39;onglet **Notifications** en haut et cliquez sur **Préférences**.
* Sélectionnez la langue de votre choix et cliquez sur **Enregistrer**.

>[!NOTE]
>
>Vous devez vous déconnecter et vous reconnecter à [!DNL Journey Optimizer] pour appliquer les modifications.

## Rechercher et filtrer{#section_lgm_hpz_pgb}

Dans la plupart des listes, une barre de recherche vous permet de rechercher un élément.

Vous pouvez accéder aux **[!UICONTROL Filtres]** en cliquant sur l&#39;icône de filtre en haut à gauche de la liste. Le menu Filtrer vous permet de filtrer les éléments affichés selon différents critères. Vous pouvez choisir d’afficher uniquement les éléments d’un certain type ou statut, ceux que vous avez créés ou ceux que vous avez modifiés au cours des 30 derniers jours.

Dans la liste de parcours, outre les **[!UICONTROL filtres de création]**, vous pouvez également filtrer les parcours affichés selon leur état et leur version (**[!UICONTROL filtres d’état et de version]**). Vous pouvez choisir d’afficher uniquement les parcours qui utilisent un événement, un groupe de champs ou une action spécifique (**[!UICONTROL filtres d’Activité]** et **[!UICONTROL filtres de données]**). Les **[!UICONTROL filtres de publication]** vous permettent de sélectionner une date de publication ou un utilisateur. Vous pouvez, par exemple, choisir d’afficher les dernières versions des parcours en direct qui ont été publiées hier. [En savoir plus](building-journeys/using-the-journey-designer.md).

>[!NOTE]
>
>Notez que les colonnes affichées peuvent être personnalisées à l&#39;aide du bouton de configuration situé en haut à droite des listes. La personnalisation est enregistrée pour chaque utilisateur.

Utilisez les colonnes **[!UICONTROL Dernière mise à jour]** et **[!UICONTROL Dernière mise à jour par]** pour vérifier quand s&#39;est produit la dernière mise à jour de vos parcours et qui l&#39;a enregistrée.

![](assets/journey74.png)

Dans les volets de configuration Événement, Source de données et Action, le champ **[!UICONTROL Utilisé dans]** affiche le nombre de parcours qui utilisent ce événement, ce groupe de champs ou cette action particulier. Vous pouvez cliquer sur le bouton **[!UICONTROL parcours de Vue]** pour afficher la liste des parcours correspondants.

![](assets/journey3bis.png)

Dans les différentes listes, vous pouvez exécuter des actions de base sur chaque élément. Par exemple, vous pouvez duplicata ou supprimer un élément.

![](assets/journey4.png)

## Parcourir les champs Adobe Experience Platform {#friendly-names-display}

Lors de la définition de [charge utile du événement](event/about-creating.md#define-the-payload-fields), [charge utile du groupe de champs](datasource/configure-data-sources.md#define-field-groups) et de la sélection de champs dans l’[éditeur d’expressions](https://experienceleague.adobe.com/docs/journeys/using/building-advanced-conditions-journeys/expressionadvanced.html), le nom d’affichage s’affiche en plus du nom du champ. Ces informations sont récupérées à partir de la définition de schéma dans le modèle de données d’expérience.

Si des descripteurs tels que &quot;xdm:alternativeDisplayInfo&quot; sont fournis lors de la configuration de schémas, les noms conviviaux remplaceront les noms d’affichage. Elle est particulièrement utile lorsque vous utilisez des &quot;eVars&quot; et des champs génériques. Vous pouvez configurer des descripteurs de nom conviviaux via un appel d’API. Pour plus d&#39;informations, consultez le [Schéma Registry developer guide](https://experienceleague.adobe.com/docs/experience-platform/xdm/api/getting-started.html).

![](assets/xdm-from-descriptors.png)

Si un nom convivial est disponible, le champ s’affiche sous la forme `<friendly-name>(<name>)`. Si aucun nom convivial n’est disponible, le nom d’affichage s’affiche, par exemple `<display-name>(<name>)`. Si aucun d&#39;eux n&#39;est défini, seul le nom technique du champ s&#39;affiche `<name>`.

>[!NOTE]
>
>Les noms conviviaux ne sont pas récupérés lorsque vous sélectionnez des champs à partir d’une union de schémas.

## Accessibilité{#cjm-accessibility}

Voici les différents raccourcis disponibles dans l&#39;interface de [!DNL Journey Optimizer].

_Dans la liste des parcours, actions, sources de données ou événements :_

* Appuyez sur **c** pour créer un parcours, une action, une source de données ou un événement.

_Lors de la configuration d’une activité dans un parcours :_

La trame est automatiquement enregistrée. Vous pouvez voir, en haut à gauche de la trame, l&#39;état d&#39;enregistrement.

* Appuyez sur **escape** pour fermer le volet de configuration et ignorer les modifications apportées. Il s’agit de l’équivalent du bouton **[!UICONTROL Annuler]**.
* Appuyez sur **Entrée** ou cliquez en dehors du volet pour fermer le volet de configuration. Les modifications sont enregistrées. Il s’agit de l’équivalent du bouton **[!UICONTROL Ok]**.
* Si vous appuyez sur **Supprimer** ou **backspace**, vous pouvez ensuite appuyer sur **Entrée** pour confirmer la suppression.

_Dans les fenêtres contextuelles :_

* Appuyez sur **escape** pour le fermer (équivalent du bouton **[!UICONTROL Annuler]**).
* Appuyez sur **Entrée** pour enregistrer ou confirmer (équivalent du bouton **[!UICONTROL Ok]** ou **[!UICONTROL Enregistrer]**).

_Dans le volet de configuration événement, source de données ou action :_

* Appuyez sur **escape** pour fermer le volet de configuration sans enregistrer.
* Appuyez sur **Entrée** pour enregistrer les modifications et fermer le volet de configuration.
* Appuyez sur **tab** pour passer d’un champ à l’autre pour configurer.

_Dans l’éditeur d’expressions simple_

* Cliquez doublon sur un champ, à gauche, pour ajouter une requête (équivalent à glisser-déposer).

_Lorsque vous parcourez les champs XDM :_

* Si vous cochez &quot;noeud&quot;, tous les champs du noeud seront sélectionnés.

_Dans toutes les zones de texte :_

* Utilisez la combinaison de touches **Ctrl/Commande + A** pour sélectionner le texte. Dans la prévisualisation de charge utile, elle sélectionne la charge.

_Dans un écran avec une barre de recherche :_

* Utilisez la combinaison de touches **Ctrl/Commande + F** pour sélectionner la barre de recherche.

_Dans la trame d&#39;un parcours :_

* Utilisez la combinaison de touches **Ctrl/Commande + A** pour sélectionner toutes les activités.
* Si une ou plusieurs activités sont sélectionnées, appuyez sur **Supprimer** ou **backspace** pour les supprimer. Ensuite, vous pouvez appuyer sur **Entrée** pour confirmer dans la fenêtre contextuelle de confirmation.
* Cliquez en doublon sur une activité de la palette de gauche pour l’ajouter à la première position disponible (de haut en bas).

_Dans un parcours :_

* Appuyez sur **T** pour activer/désactiver le mode de test.
* Dans un parcours basé sur un événement en mode test, appuyez sur **E** pour déclencher un événement.
* Dans un parcours basé sur les segments, lorsque l&#39;option **profil unique à la fois** est sélectionnée en mode test, appuyez sur **P** pour déclencher un événement.
* En mode test, appuyez sur **L** pour afficher les journaux.
