# Utiliser des API [!DNL Journey Optimizer] {#apis-gs}

## Accès rapide {#quick-access}

Consultez la [référence complète des API](https://developer.adobe.com/journey-optimizer-apis){target="_blank"} pour accéder à toutes les API Journey Optimizer et les essayer directement. Pour commencer, assurez-vous de [configurer l’authentification](https://developer.adobe.com/journey-optimizer-apis/references/authentication){target="_blank"} pour obtenir les informations d’identification requises.

## Vue d’ensemble {#overview}

L’API Adobe Journey Optimizer vous permet de proposer des expériences clients personnalisées, connectées et opportunes sur n’importe quel appareil, canal ou application, ce qui vous permet de gérer efficacement le parcours client de bout en bout. Le parcours client est l’ensemble du processus d’interaction d’une cliente ou d’un client avec la marque, depuis le premier contact jusqu’au départ de la cliente ou du client. Il débute avec la phase de sensibilisation, où le client ou la cliente découvre la marque et commence à interagir. Le client ou la cliente interagira ensuite davantage avec la marque, se rendra sur les sites web et physiques, effectuera des achats, enverra des messages ou publiera des commentaires.

Adobe Journey Optimizer est conçu de manière native sur Adobe Experience Platform et associe un profil client en temps réel unifié, un framework ouvert fondé sur l’API, une prise de décisions sur les offres centralisée, une intelligence artificielle (IA) et du machine learning (ML) pour la personnalisation et l’optimisation. L’intégration à l’API Journey Optimizer permet aux marques de déterminer intelligemment la meilleure interaction possible, avec l’échelle, la vitesse et la flexibilité adaptées, pour l’ensemble du parcours client.

**Commencer avec les API Journey Optimizer :**

* **[Consulter la référence complète des API](https://developer.adobe.com/journey-optimizer-apis){target="_blank"}** : accédez à toutes les API Journey Optimizer et testez-les directement.
* **[Configurer l’authentification](https://developer.adobe.com/journey-optimizer-apis/references/authentication){target="_blank"}** : obtenez les informations d’identification requises pour commencer à utiliser les API.
* **[API Decision Management](../offers/api-reference/getting-started.md)** : gérez les offres et les décisions par programme.
* **[API Experience Decisioning](../experience-decisioning/api-reference/getting-started.md)** : proposez des éléments de décision personnalisés à l’aide d’expériences basées sur du code.

Journey Optimizer propose des API dédiées à la gestion des décisions, ce qui vous permet de gérer les offres, les décisions et les emplacements par programme.

Consultez le [guide de développement des API Decision Management](../offers/api-reference/getting-started.md) pour commencer à utiliser les API de prise de décision sur les offres.

## Api Experience Decisioning {#experience-decisioning-apis}

Journey Optimizer fournit également des API Experience Decisioning afin que vous puissiez proposer des éléments de décision personnalisés à l’aide d’expériences basées sur du code. La fonctionnalité de décisions pour les expériences offre une approche simplifiée de la personnalisation avec des éléments de décision, des règles d’éligibilité et des stratégies de sélection.

**Opérations d’API disponibles :**

* **Éléments de décision** : créez, lisez, mettez à jour et supprimez des éléments de décision.
* **Stratégies de sélection** : définissez comment sélectionner et classer les éléments de décision.
* **Règles d&#39;éligibilité** : définissez les conditions d’éligibilité des éléments.
* **Collections d’éléments** : regroupez les éléments de décision en collections.
* **Formules de classement** : configurez une logique de classement personnalisée.
* **Emplacements** : définissez où les éléments de décision peuvent apparaître.

Pour en savoir plus, consultez la [référence des API Experience Decisioning](../experience-decisioning/api-reference/getting-started.md) et découvrez comment [diffuser des offres à l’aide d’expériences basées sur du code](../experience-decisioning/gs-experience-decisioning.md).
